Scouting Application (Project Four)

Project Description

The idea for this project was based of a gap in the market that I identified when working in the football industry. The web application enables scouts to have a centralised location where they can write and store player reports to keep track of their expenses. The project makes use of two third party APIs, Google Maps in order to calculate the expenses and ChartJS to give a better visual representation of the data to the user.

Moodfiy (Project Three)

Project Description

During a course module on APIs, I became interested in how APIs can simplify very complex tasks and as a result my group decided to use an obscure API for this project. We decided to use Face++, a facial recognition API that returns data based on a picture of a face. When researching Face++ we noticed it returned a mood based on the picture uploaded. We decided that as Spotify also had a mood category for their playlists we could create an application that took the mood of a person's face and return them a relevant Spotify playlist.

This was a RESTful application and made use of Spotify Oauth to allow people to play the full song from the playlists returned. I am aware that the collaborative focus of this project is an important preparation for working within a professional team based environment.

Stadia (Project Two)

Project Description

Stadia was a stadium review website which allowed users to create a new stadium to review or add reviews to a stadium that was already in the database. Stadia was a fully RESTful EJS application and was styled using Materialize.

Two Player Hangman (Project One)

Project Description

The first project of the course, required me to create a game to put into practice the three weeks of JavaScript we had been taught. I decided to create a version of Hangman that you could play against a friend. This is a turn based game and each user gets a point for each letter they guess correctly. The person with the most points wins.

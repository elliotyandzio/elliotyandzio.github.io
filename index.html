Project Description

The idea for this project was based on a gap in the market that I identified when working in the football industry. The web application enables scouts to have a centralised location where they can write and store player reports to keep track of their expenses. The project makes use of two third party APIs, Google Maps in order to calculate the expenses and ChartJS to give a better visual representation of the data to the user.

Project Description

During a course module on APIs, I became interested in how APIs can simplify very complex tasks and as a result my group decided to use an obscure API for this project. We decided to use Face++, a facial recognition API that returns data based on a picture of a face. When researching Face++ we noticed it returned a mood based on the picture uploaded. As Spotify has a mood category for their playlists, we decided to create an application which would create a Spotify playlist based on a person’s mood. This was a RESTful application and made use of Spotify Oauth to allow people to play the full song from the playlists returned. The collaborative focus of this project was an important preparation for working within a professional team based environment.
